# Phase 02 Plan 01: Agent Infrastructure & Candidate Hunting Setup Summary

**Status:** COMPLETE

**One-liner:** Built reusable BaseAgent pattern with Anthropic SDK integration and data models (HuntingSession, TargetCompany, Candidate, OutreachDraft) for AI-powered candidate hunting.

## What Was Built

### 1. **Data Models (src/app/models.py)**

Added four new SQLModel tables with complete field definitions and validation:

- **HuntingSession:** Represents a candidate hunting run per job
  - Fields: id, job_id (FK), status (HuntingStatus enum), created_at, updated_at, started_at, completed_at
  - Status values: pending, in_progress, completed, failed

- **TargetCompany:** AI-identified target companies
  - Fields: id, hunting_session_id (FK), name, industry, size_description, linkedin_url (optional), created_at
  - Mockable: These will be generated by CompanyMapperAgent or added manually

- **Candidate:** Discovered candidates at target companies
  - Fields: id, hunting_session_id (FK), target_company_id (FK), name, current_title, current_company, email (optional), linkedin_url (optional), status (CandidateStatus enum), created_at, updated_at
  - Status values: hunting, approved, rejected

- **OutreachDraft:** AI-drafted messages per candidate
  - Fields: id, candidate_id (FK), job_id (FK), linkedin_message (10K char max), email_message (10K char max), status (OutreachStatus enum), created_at, updated_at, sent_at (optional)
  - Status values: pending_review, approved, sent, replied, not_interested, error

All enums use lowercase single-word values (e.g., `pending_review` not `pending-review`). All timestamps use `datetime.utcnow` factory defaults. Foreign keys are indexed.

### 2. **Agent Infrastructure (src/app/agents/)**

**BaseAgent** (`base.py`, 123 lines)
- Abstract base class for all agents using Anthropic SDK
- Constructor initializes `anthropic.AsyncClient()` with API key from `ANTHROPIC_API_KEY` env var
- Method signatures:
  - `system_prompt()`: Returns system instruction string
  - `tools()`: Returns list of tool definitions (OpenAPI/JSON schema format)
  - `parse_response(tool_results)`: Parses Claude tool responses into domain models
  - `async run(request)`: Main method that calls Claude API and handles responses
- Error handling: Catches `anthropic.APIError` gracefully, returns error status without re-raising
- Returns `AgentResponse` (status, data, error) for all responses

**CompanyMapperAgent** (`company_mapper.py`, 163 lines)
- Identifies 5-10 target companies matching job criteria via Claude API
- Request model: `CompanyMapperRequest` (job_title, job_description, target_industries, target_company_types)
- Response model: `CompanyMapperResponse` (list of TargetCompanyItem)
- Tool definition: `identify_companies` with JSON schema for structured output
- Mock company fallback: Hardcoded list of 5 realistic companies (Stripe, Notion, Figma, McKinsey, Airbnb) with clearly marked comment `# MOCK DATA`
- Gracefully falls back to mock data if API fails or returns invalid responses

**Module Exports** (`__init__.py`)
- Exports CompanyMapperAgent for public API

## Key Decisions Made

1. **Explicit `__tablename__` on SQLModel classes** — Required for proper foreign key resolution with SQLAlchemy 2.x
2. **Mock company fallback** — Enables testing without API calls; clearly marked for future real API integration
3. **AgentResponse standard format** — Consistent error handling across all agents (status, data, error fields)
4. **Async/await throughout** — Enables concurrent agent execution in FastAPI
5. **No LangChain** — Direct Anthropic SDK usage per project standards (simpler, fewer dependencies)
